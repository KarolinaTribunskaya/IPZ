## 4. Типи даних
Оголосимо змінні у програмі. Для цогоо застосуємо ключове слово `var` далі необхідно написати ім'я змінної. Наприклад:
```js
var num = 1.45;
var Num = 23;
var string = "Hello";
var bool = true;
```
Ми можемо побачити, що ця мова програмування чутлива до зміни регістру й змінні num і Num - це незадлежні змінні й вони можуть набувати різних значень. 
Також JavaScript - мова програмування з динамічною типізацією. Це означає, що ми можмо присвоювати змінній будь-яке значення не вказуючи його тип. Наприклад, число з плаваючою точкою, цілочисельне, строкове, булеве значення і т.д.

Існує новий формат оголошення змінної, який більш поширений зараз. Для створення змінної використовуюсь слово `let`. Наприклад:
```js
let age = 18;
```

Щоб оголосити константну, тобто, змінну, значення якої нем можна змінити протягом роботи програми, використовуйте const замість let:
```js
const Birthday = '15.03.2002';
```

## 6 Математичні операціїї
У JavaScript використовують такі самі операції, як і в інших мовах програмування: додавання, віднімання, ділення, множення ... Приклади:
```js
//Оголошення змінних і присвоєння їм значення
var firstNum = 24;
var secondNum = 12;

//Додавання
document.write(firstNum + "+" + secondNum + " = ", firstNum + secondNum, "<br>");
//Віднімання
document.write(firstNum + "-" + secondNum + " = ", firstNum - secondNum, "<br>");
//Ділення
document.write(firstNum + "/" + secondNum + " = ", firstNum / secondNum, "<br>");
//Залишок від ділення
document.write(firstNum + "%" + secondNum + " = ", firstNum % secondNum, "<br>");
```
Операції округлення й використання бібліотеки Math для отримання математичних констант й функцій:
```js
//Використання Math для отримання абсолютного значення
document.write("Абсолютне значення ("+ -12 + ")" + " = ", Math.abs(-12), "<br>");
//Значення числа пі та експоненти
document.write("pi = ", Math.PI, "<br>");
document.write("e = ", Math.E + "<br>");

//Округлення числа різними способами
var number = 2.3; 
//Округлення до більшого
document.write("Округлення до більшого " + number + " ~ ", Math.ceil(number) + "<br>");
//Округлення до меншого
document.write("Округлення до меншого " + number + " ~ ", Math.floor(number) + "<br>");
//Округлення математичне
document.write("Округлення математичне " + number + " ~ ", Math.round(number) + "<br>");
```

## 8 Цикли
При написанні скриптів часто постає завдання зробити однотипне дію багато разів.

Наприклад, вивести товари зі списку один за іншим. Або просто перебрати всі числа від 1 до 10 і для кожного виконати однаковий код.

Для багаторазового повторення однієї ділянки коду передбачені цикли.

Цикл «while»
Цикл while має наступний синтаксис:
```js
while (condition) {
  // код
  // також званий "тілом циклу"
}
```
Код з тіла циклу виконується, поки умова condition істинно.

Наприклад, цикл нижче виводить i, поки i <3:
```js
let i = 0;
while (i <3) {// виводить 0, потім 1, потім 2
  alert (i);
  i ++;
}
```
Одне виконання тіла циклу називається ітерація. Цикл в прикладі вище робить три ітерації.

Якби рядок i ++ була відсутня в прикладі вище, то цикл б повторювався (в теорії) нескінченно. На практиці, звичайно, браузер не дозволить такому трапитися, він надасть користувачеві можливість зупинити скрипт, а JavaScript на стороні сервера доведеться зупинити процес.

Будь-яке вираження або змінна може бути умовою циклу, а не тільки порівняння: умова while обчислюється і перетворюється в логічне значення.

Наприклад, while (i) - більш короткий варіант while (i! = 0):
```js
let i = 3;
while (i) {// коли i дорівнюватиме 0, умова стане хибним, і цикл зупиниться
  alert (i);
  i--;
}
```
Фігурні дужки не потрібні для тіла циклу з одного рядка
Якщо тіло циклу складається лише з однієї інструкції, ми можемо опустити фігурні дужки {...}:
```js
let i = 3;
while (i) alert (i--);
```
Цикл «do ... while»
Перевірку умови можна розмістити під тілом циклу, використовуючи спеціальний синтаксис do..while:
```js
do {
  // тіло циклу
} While (condition);
```
Цикл спочатку виконає тіло, а потім перевірить умова condition, і поки його значення дорівнює true, він буде виконуватися знову і знову.

наприклад:
```js
let i = 0;
do {
  alert (i);
  i ++;
} While (i <3);
```
Така форма синтаксису виправдана, якщо ви хочете, щоб тіло циклу виповнилося хоча б один раз, навіть якщо умова виявиться помилковим. На практиці частіше використовується форма з передумовою: while (...) {...}.

Цикл «for»
Більш складний, але при цьому найпоширеніший цикл - цикл for.

Виглядає він так:

for (початок; умова; крок) {
  // ... тіло циклу ...
}
Давайте розберемося, що означає кожна частина, на прикладі. Цикл нижче виконує alert (i) для i від 0 до (але не включаючи) 3:

for (let i = 0; i <3; i ++) {// виведе 0, потім 1, потім 2
  alert (i);
}
Розглянемо конструкцію for:

частина
початок i = 0 Виконується один раз при вході в цикл
умова i <3 Перевіряється перед кожною итерацией циклу. Якщо воно обчислити в false, цикл зупиниться.
крок i ++ Виконується після тіла циклу на кожній ітерації перед перевіркою умови.
тіло alert (i) Виконується знову і знову, поки умова обчислюється в true.

## 9 Умовні оператори
Іноді нам потрібно виконати різні дії в залежності від умов.

Для цього ми можемо використовувати інструкцію if і умовний оператор?, Який також називають оператором «знак питання».

Інструкція «if»
Інструкція if (...) обчислює умова в дужках і, якщо результат true, то виконує блок коду.

наприклад:
```js
let year = prompt ( 'В якому році була опублікована специфікація ECMAScript-2015 році?', '');

if (year == 2015) alert ( 'Ви маєте рацію!');
```
В наведеному вище прикладі, умова - це проста перевірка на рівність (year == 2015), але воно може бути і набагато більш складним.

Якщо ми хочемо виконати більше однієї інструкції, то потрібно укласти блок коду в фігурні дужки:
```js
if (year == 2015) {
  alert ( "Правильно!");
  alert ( "Ви такий розумний!");
}
```
Ми рекомендуємо використовувати фігурні дужки {} завжди, коли ви використовуєте інструкцію if, навіть якщо виконується тільки одна команда. Це покращує читабельність коду.

Перетворення до логічного типу
Інструкція if (...) обчислює вираз в дужках і перетворює результат до логічного типу.

Давайте згадаємо правила перетворення типів з голови Перетворення типів:

Число 0, порожній рядок "", null, undefined і NaN стають false.
Решта значення стають true.
Таким чином, код за такої умови ніколи не виконається:
```js
if (0) {// 0 is falsy
  ...
}
```
... а при такому - виконається завжди:
```js
if (1) {// 1 is truthy
  ...
}
```
Ми також можемо передати заздалегідь обчислена в змінної логічне значення в if, наприклад так:
```js
let condition = (year == 2015); // перетвориться до true або false

if (condition) {
  ...
}
```
Блок «else»
Інструкція if може містити необов'язковий блок «else» ( «інакше»). Він виконується, коли умова помилкова.

наприклад:
```js
let year = prompt ( 'В якому році була опублікована специфікація ECMAScript-2015 році?', '');

if (year == 2015) {
  alert ( 'Так ви знавець!');
} Else {
  alert ( 'А ось і неправильно!'); // будь-яке значення, крім 2015
}
```
Кілька умов: «else if»
Іноді, потрібно перевірити кілька варіантів умови. Для цього використовується блок else if.

наприклад:
```js
let year = prompt ( 'В якому році була опублікована специфікація ECMAScript-2015 році?', '');

if (year <2015) {
  alert ( 'Це дуже рано ...');
} Else if (year> 2015) {
  alert ( 'Це запізно');
} Else {
  alert ( 'Вірно!');
}
```
У наведеному вище коді JavaScript спочатку перевірить year <2015. Якщо це не так, він переходить до наступного умові year> 2015. Якщо воно теж помилково, тоді спрацює останній alert.

Блоків else if може бути і більше. Присутність блоку else не є обов'язковим.

Умовний оператор "?"
Іноді нам потрібно визначити змінну в залежності від умови.

наприклад:
```js
let accessAllowed;
let age = prompt ( 'Скільки вам років?', '');

if (age> 18) {
  accessAllowed = true;
} Else {
  accessAllowed = false;
}

alert (accessAllowed);
```

# 10 Конструкція "switch"
Конструкція switch замінює собою відразу кілька if. Вона являє собою більш наочний спосіб порівняти вираз відразу з декількома варіантами. 
## Cинтаксис
Конструкція switch має один або більше блоків case і необов'язковий блок default.

Виглядає вона так:
```js
switch (x) {
  case 'value1': // if (x === 'value1')
    ...
    [Break]

  case 'value2': // if (x === 'value2')
    ...
    [Break]

  default:
    ...
    [Break]
}
```
Змінна x перевіряється на рівність першому значенню value1, потім другого value2 і так далі.
Якщо відповідність встановлено - switch починає виконуватися від відповідної директиви case і далі, до найближчого break (або до кінця switch).
Якщо жоден case не співпали - виконується (якщо є) варіант default.
##Приклад роботи
Приклад використання switch:
```js
let a = 2 + 2;

switch (a) {
  case 3:
    alert ( 'Малувато');
    break;
  case 4:
    alert ( 'В точку!');
    break;
  case 5:
    alert ( 'Перебір');
    break;
  default:
    alert ( "Немає таких значень");
}
```
Тут оператор switch послідовно порівняє a з усіма варіантами з case.

Спочатку 3, потім - так як немає збігу - 4. Збіг знайдено, буде виконаний цей варіант, з рядка alert ( 'В точку!') І далі, до найближчого break, який перерве виконання.

Якщо break немає, то виконання піде нижче за наступними case, при цьому інші перевірки ігноруються.

Приклад без break:
```js
let a = 2 + 2;

switch (a) {
  case 3:
    alert ( 'Малувато');
  case 4:
    alert ( 'В точку!');
  case 5:
    alert ( 'Перебір');
  default:
    alert ( "Немає таких значень");
}
```
В наведеному вище прикладі послідовно виконуватися три alert:
```js
alert ( 'В точку!');
alert ( 'Перебір');
alert ( "Немає таких значень");
```
Будь-яке вираження може бути аргументом для switch / case
І switch і case допускають будь-який вираз в якості аргументу.

наприклад:
```js
let a = "1";
let b = 0;

switch (+ a) {
  case b + 1:
    alert ( "Виконати, тому що значенням + a буде 1, що в точності так само b + 1");
    break;

  default:
    alert ( "Це не виконається");
}
```
У цьому прикладі вираз + a обчислюється в 1, що збігається з виразом b + 1 в case, і отже, код в цьому блоці буде виконаний.

## Угруповання «case»
Кілька варіантів case, що використовують один код, можна групувати.

Для прикладу, виконаємо один і той же код для case 3 і case 5, згрупувавши їх:
```js
let a = 2 + 2;

switch (a) {
  case 4:
    alert ( 'Правильно!');
    break;

  case 3: // (*) групуємо обидва case
  case 5:
    alert ( 'Неправильно!');
    alert ( "Може вам відвідати урок математики?");
    break;

  default:
    alert ( 'Результат виглядає трохи дивно. Чесно.');
}
```
Тепер обидва варіанти 3 і 5 виводять одне повідомлення.

Можливість групувати case - це побічний ефект того, як switch / case працює без break. Тут виконання case 3 починається з рядка (*) і триває в case 5, тому що відсутня break.

Тип має значення
Потрібно відзначити, що перевірка на рівність завжди сувора. Значення повинні бути одного типу, щоб виконувалося рівність.

Для прикладу, давайте розглянемо наступний код:
```js
let arg = prompt ( "Введіть число?");
switch (arg) {
  case '0':
  case '1':
    alert ( 'Один або нуль');
    break;

  case '2':
    alert ( 'Два');
    break;

  case 3:
    alert ( 'Ніколи не виконається!');
    break;
  default:
    alert ( 'Невідоме значення');
}
```
Для '0' і '1' виконається перший alert.
Для '2' - другий alert.
Але для 3, результат виконання prompt буде рядок "3", яка не відповідає суворому рівності === з числом 3. Таким чином, ми маємо «мертвий код» в case 3! Виконається варіант default.

# 11 Функції
## Що таке функція
Функція - це певний фрагмент коду, який можна описати один раз, а потім викликати на виконання в різних частинах програми якесь завгодно число раз.
Це класичне призначення функції. У цьому сценарії код, який у нас повторюється кілька разів на сторінці, ми виносимо в функцію, а потім використовуємо її в тих місцях програми, в яких нам необхідно його виконати.

Ще один варіант традиційного застосування функцій - це коли ми їх використовуємо в якості основних будівельних блоків програми. В цьому випадку поставлена ​​задача розбивається на підзадачі, а потім кожну з них ми вирішуємо за допомогою створення окремих функцій. Після цього ми зазвичай переходимо до розробки центрального коду, використовуючи в ньому все раніше розроблені функції.

В результаті така програма стає більш структурованою. У неї більше просто вносити різні зміни і додавати нові можливості.

В JavaScript створити функцію можна різними способами. це:

Function Declaration;
Function Expression;
Arrow Function;
Для початку ми розглянемо спосіб створення функції за допомогою Function Declaration. Function Declaration - це «класичний» варіант оголошення функції.

## Оголошення і виклик функції
Операції з функцією в JavaScript можна розділити на 2 етапи:

оголошення (створення) функції;
виклик (виконання) цієї функції.
1. Оголошення функції. Написання функції за допомогою Function Declaration починається з написання ключового слова function. Після цього вказується ім'я функції, круглі дужки в яких при необхідності перераховуються через кому параметри і код функції, укладений у фігурні дужки.
```js
function ім'я (параметри) {
  // код функції
}
```
наприклад:
```js
// оголошення функції someName
function someName () {
  alert ( 'Ви викликали функцію someName!');
}

// function - ключове слово, яке означає що ми хочемо створити функцію
// someName - ім'я функції
// () - круглі дужки всередині яких при необхідності ми можемо помістити параметри через кому
// {...} - код або тіло функції
```
 
## Синтаксис оголошення функції
При складанні імені функції необхідно керуватися темі ж правилами, що і при створенні імені змінної. Тобто воно може містити літери, цифри (0-9), знаки «$» і «_». Як букв рекомендується використовувати тільки букви англійського алфавіту (a-z, A-Z). Ім'я функції, також як і ім'я змінної не може починатися з цифри.

Параметрів у функції може бути скільки завгодно багато чи не бути взагалі. Круглі дужки в будь-якому випадку вказуються. Якщо параметрів кілька, то їх між собою необхідно розділити за допомогою коми. Вони дозволяють більш зручно (по імені) отримати передані аргументи функції при її виклику.

Код функції - це набір інструкцій, укладений у фігурні дужки, які необхідно виконати при її виклику. Код функції називають ще її тілом.

2. Виклик функції. Оголошена функція сама по собі не виконується. Для того щоб функцію запустити, її необхідно викликати. Виклик функції здійснюється за допомогою вказівки її імені та двох круглих дужок. Усередині дужок при необхідності їй можна передати аргументи (додаткові дані) відокремлюючи їх один від одного за допомогою коми.
```js
// виконання функції, наведеної в попередньому прикладі (без передачі їй аргументів)
someName ();
```
## Синтаксис виклику функції
Параметри і аргументи функції
Параметри функції - це локальні змінні функції, які визначаються на етапі оголошення функції в круглих дужках. Звернутися до параметрів можна тільки всередині функції, зовні вони не доступні.

Аргументи функції - це значення передані в функцію при її виклику.

наприклад:
```js
// оголошення функції sayWelcome (userFirstName і userLastName - це параметри)
// userFirstName - буде приймати значення 1 аргументу, а userLastName відповідно значення 2 аргументу
function sayWelcome (userFirstName, userLastName) {
  console.log ( 'Ласкаво просимо,' + userLastName + '' + userFirstName);
}

// виклик функції sayWelcome ( 'Іван' і 'Іванов' - це аргументи)
sayWelcome ( 'Іван', 'Іванов');

// ще один виклик функції sayWelcome ( 'Петро', 'Петров' - це аргументи)
sayWelcome ( 'Петро', 'Петров');
```
Таким чином, параметри - це один із способів в JavaScript, за допомогою якого можна отримати аргументи функції.


 
В JavaScript при виконанні функції кількість аргументів не обов'язково має збігатися з кількістю параметрів. Якщо функції не був переданий аргумент, який ми хочемо отримати за допомогою параметра, то в цьому випадку він буде мати значення undefined.

Наприклад, викличемо функцію з прикладу, наведеного вище, без передачі їй одного аргументу:
```js
// виклик функції sayWelcome з одним аргументом 'Петро'
sayWelcome ( 'Петро'); // Ласкаво просимо, undefined Петро
Виклик функції sayWelcome без аргументів:

// виклик функції sayWelcome без аргументів
sayWelcome (); // Ласкаво просимо, undefined undefined
```
# 12 Робота з об'єктами
JavaScript спроектований на основі простої парадигми. В основі концепції лежать прості об'єкти. Об'єкт - це набір властивостей, і кожне властивість складається з імені та значення, асоційованого з цим ім'ям. Значенням властивості може бути функція, яку можна назвати методом об'єкта. На додаток до вбудованим в браузер об'єктів, ви можете визначити свої власні об'єкти. Ця глава описує як користуватися об'єктами, властивостями, функціями і методами, а також як створювати свої власні об'єкти.

## Oгляд об'єктів
Об'єкти в JavaScript, як і в багатьох інших мовах програмування, схожі на об'єкти реального життя. Концепцію об'єктів JavaScript легше зрозуміти, проводячи паралелі з реально існуючими в житті об'єктами.

В JavaScript об'єкт - це самостійна одиниця, що має властивості і певний тип. Порівняємо, наприклад, з чашкою. У чашки є колір, форма, вага, матеріал, з якого вона зроблена, і т.д. Точно так же, об'єкти JavaScript мають властивості, які визначають їх характеристики.

## Об'єкти і властивості
В JavaScript об'єкт має властивості, асоційовані з ним. Властивість об'єкта можна розуміти як змінну, закріплену за об'єктом. Властивості об'єкта по суті є тими ж самими змінними JavaScript, за тим винятком, що вони закріплені за об'єктом. Властивості об'єкта визначають його характеристики. Отримати доступ до властивості об'єкта можна за допомогою точкового записи:

```js
objectName.propertyName
```
Як і всі змінні JavaScript, ім'я об'єкта (яке теж може бути змінної) і ім'я властивості є чутливими до регістру. Ви можете визначити властивість вказавши його значення. Наприклад, давайте створимо об'єкт myCar і визначимо його властивості make, model, і year наступним чином:
```js
var myCar = new Object ();
myCar.make = "Ford";
myCar.model = "Mustang";
myCar.year = 1969;
Невизначені властивості об'єкта є undefined (а не null).

myCar.color; // undefined
```
Властивості об'єктів JavaScript також можуть бути доступні або задані з використанням скобочной записи (детальніше див. Property accessors). Об'єкти іноді називаються асоціативними масивами, оскільки кожне властивість пов'язана зі строковим значенням, яке можна використовувати для доступу до нього. Так, наприклад, ви можете отримати доступ до властивостей об'єкта myCar наступним чином:
```js
myCar [ "make"] = "Ford";
myCar [ "model"] = "Mustang";
myCar [ "year"] = 1969;
```
Імена властивостей об'єкта можуть бути рядками JavaScript, або тим, що може бути конвертувати в рядок, включаючи порожню рядок. Як би там не було, доступ до будь-якого імені властивості, яке містить невалідний JavaScript ідентифікатор (наприклад, ім'я властивості містить в собі прогалину і тире або починається з цифри), може бути отриманий з використанням квадратних дужок. Цей спосіб запису також корисний, коли імена властивостей повинні бути динамічно визначені (коли ім'я властивості не визначено до моменту виконання). Приклади далі:
```js
var myObj = new Object (),
    str = "myString",
    rand = Math.random (),
    obj = new Object ();

myObj.type = "Dot syntax";
myObj [ "date created"] = "String with space";
myObj [str] = "String value";
myObj [rand] = "Random Number";
myObj [obj] = "Object";
myObj [ ""] = "Even an empty string";

console.log (myObj);
```
Зверніть увагу, що всі ключі з квадратними дужками перетворюються в тип String, оскільки об'єкти в JavaScript можуть мати в якості ключа тільки тип String. Наприклад, у наведеному вище коді, коли ключ obj додається в myObj, JavaScript викликає метод obj.toString () і використовує цю результуючий рядок в якості нового ключа.

Ви також можете отримати доступ до властивостей, використовуючи значення рядка, яке зберігається в змінної:
```js
var propertyName = "make";
myCar [propertyName] = "Ford";

propertyName = "model";
myCar [propertyName] = "Mustang";
```
Ви можете користуватися квадратними дужками в конструкції for ... in щоб виконати ітерацію всіх властивостей об'єкта, для яких вона дозволена. Щоб показати як це працює, наступна функція показує всі властивості об'єкта, коли ви передаєте в неї сам об'єкт і його ім'я як аргументи функції:
```js
function showProps (obj, objName) {
  var result = "";
  for (var i in obj) {
    if (obj.hasOwnProperty (i)) {
        result + = objName + "." + I + "=" + obj [i] + "\ n";
    }
  }
  return result;
}
```
Так що якщо викликати цю функцію ось так showProps (myCar, "myCar"), то отримаємо результат:
```js
myCar.make = Ford
myCar.model = Mustang
myCar.year = 1969
```

## Використання функції конструктора
Інший спосіб створити об'єкт в два кроки описаний нижче:

Визначте тип об'єкта, написавши функцію-конструктор. Назва такої функції, як правило, починається з великої літери.
Створіть екземпляр об'єкта за допомогою ключового слова new.
Щоб визначити тип об'єкта створіть функцію, яка визначає тип об'єкта, його ім'я, властивості і методи. Наприклад припустимо, що ви хочете створити тип об'єкта для опису машин. Ви хочете, щоб об'єкт цього типу називався car, і ви хочете, щоб у нього були властивості make, model, і year. Щоб зробити це, напишіть наступну функцію:
```js
function Car (make, model, year) {
  this.make = make;
  this.model = model;
  this.year = year;
}
```
Зауважте, що використовується this щоб привласнити значення (передані як аргументи функції) властивостей об'єкта.

Тепер ви можете створити об'єкт, званий mycar, в такий спосіб:
```js
var mycar = new Car ( "Eagle", "Talon TSi", 1993);
```
Ця інструкція створює об'єкт типу Car з посиланням mycar і привласнює певні значення його властивостям. Значним mycar.make стане рядок "Eagle", mycar.year - це ціле число 1993, і так далі.

Ви можете створити стільки об'єктів car, скільки потрібно, просто викликаючи new. наприклад:
```js
var kenscar = new Car ( "Nissan", "300ZX", 1992);
var vpgscar = new Car ( "Mazda", "Miata", 1990);
```
Об'єкт може мати властивість, яке буде іншим об'єктом. Наприклад, далі визначається об'єкт типу Person наступним чином:
```js
function Person (name, age, sex) {
  this.name = name;
  this.age = age;
  this.sex = sex;
}
```
і потім створити два нових примірника об'єктів Person як показано далі:
```js
var rand = new Person ( "Rand McKinnon", 33, "M");
var ken = new Person ( "Ken Jones", 39, "M");
```
Потім, ви можете переписати визначення car і включити в нього властивість owner, якому призначити об'єкт person наступним чином:
```js
function Car (make, model, year, owner) {
  this.make = make;
  this.model = model;
  this.year = year;
  this.owner = owner;
}
```
Потім, щоб створити екземпляри нових об'єктів, виконайте наступні інструкції:
```js
var car1 = new Car ( "Eagle", "Talon TSi", 1993, rand);
var car2 = new Car ( "Nissan", "300ZX", 1992, ken);
```
Зауважте, що замість того, щоб передавати рядок, літерал або ціле число при створенні нових об'єктів, у виразах вище передаються об'єкти rand і ken як аргумент функції. Тепер, якщо вам потрібно дізнатися ім'я власника car2, це можна зробити наступним чином:
```js
car2.owner
```
Зауважте, що в будь-який час ви можете додати нову властивість раніше створеного об'єкту. Наприклад, вираз
```js
car1.color = "black";
```
додає властивість color до car1, і встановлює його значення рівним "black." Як би там не було, це не впливає на будь-які інші об'єкти. Щоб додати нову властивість всіх об'єктах одного типу, ви повинні додати властивість в визначення типу об'єкта car.
